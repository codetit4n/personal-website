In this article I will be talking about my reasons to use vim.

> NOTE: This is not a tutorial for vim.

## VSCode is good

This might come as bit of a surprise to some people because more often than not people
who advocate for vim don't have good things to say about VSCode. But I think VSCode is
an amazing tool, specially for beginner programmers.

Imagine you are a new programmer and know nothing about code editors, auto-completion,
LSPs, plugins. At this point your priority is not to learn about vim motions, LSPs,
etc. You just want to hit the road running and start writing some code. VSCode is
very good tool for that. You just install it, add a few plugins and start
coding, no overhead.

## Vim is better

Lets, talk about vim now. Vim is what you use when you want to customize your coding
experience. Let's break it down:

### Vim is fun

Let's face it, coding can get pretty tedious. Vim keeps it fun. Moving around your
editor without ever touching the mouse feels pretty awesome.

### Vim motions

This is what makes vim so incredible to use. If you don't ever want to use vim I would
still suggest you to try vim motions. You can even use them in VSCode. Just try it for
fun. I bet you'll like it.

### Customization

In vim you can customize everything, and I mean everything. Now, I get there are
customizations available for VSCode also, but vim can do so much more. You can opt-out
of features that you do not like, which is not possible in VSCode. Just pick what you
like and remove all the bloat.

### My Vim configuration

I use a newer version of [vim](https://www.vim.org) called [neovim](https://neovim.io).
You can customize it with this amazing language called [Lua](https://www.lua.org). Its
easy to learn, pretty powerful and totally awesome. Now, I am not going to ramble on
about how I configured my vim, that is not the goal of this article. You can checkout
my configuration here: https://github.com/codetit4n/nvim-config

## Conclusions

Working with vim make you feel like a coding ninja. Just try it!
